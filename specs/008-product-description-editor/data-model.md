# Data Model: Rich Text Product Description

## Entities

### `Product`
- **field `description`**:
  - Type: `String` (PostgreSQL `TEXT`)
  - Semantics: Changed from plain text to arbitrary HTML content generated by TipTap.
  - Constraints: The payload size will increase but standard text storage limits in Postgres easily handle this.

## Interface Contracts

### Payload Storage
The frontend will serialize the TipTap editor content as raw HTML string (using `editor.getHTML()`) and send it in the `description` field when calling the POST/PUT `/api/products` endpoints.

### Rendering
On the consumer side (POS, Landing), the HTML string will be rendered using React's `dangerouslySetInnerHTML`. Because TipTap limits generated tags to its semantic plugins, basic usage is safe. However, adding DOMPurify or custom sanitization is a best practice on the display end. Since we are in Next.js/React, we will use a safe rendering wrapper or relying on the backend to accept only trusted inputs from the admin panel.
